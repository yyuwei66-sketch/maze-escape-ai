\section*{Abstract}

This paper introduces a survival game titled “Maze Escape”， designed to explore the practical applications of artificial intelligence search and optimization algorithms. The game simulates a scenario in which a human is chased by monsters on a 30 × 30 circular grid. It implements and evaluates five distinct AI strategies: \textbf{A* path planning, the Greedy Algorithm, Breadth-First Search, the Minimax algorithm, and the simulated Annealing Algorithm.} The maze itself is procedurally generated using a\textbf{ Genetic Algorithm}, which optimizes connectivity, wall density, path diversity, and the richness of branching paths over multiple generations of evolution. 

Through this project, we demonstrate the diverse behaviors exhibited by AI in constrained, adversarial, and real-time conditions, as well as how the choice of search strategies directly influences the game’s challenge and player experience. The results indicate that the A* algorithm enables the most direct pursuit, the Greedy Algorithm achieves high performance at low computational cost, and the Simulated Annealing Algorithm introduces controllable unpredictability. Genetic Algorithms consistently generate well-connected, navigable maps, thereby maintaining the game experience.

In the SOF Principle of Artificial Intelligence course, we explored a variety of search, planning, and optimization algorithms. Built upon these foundational course concepts, our Maze Escape project elevates static, single-agent pathfinding into a dynamic, multi-dimensional adversarial scenario by introducing procedurally evolving mazes, expanding variable dimensions, and incorporating bidirectional pursuit dynamics.

To comprehensively evaluate algorithmic behaviors, we designed two complementary gameplay modes: Escape Mode and Chase Mode. In Chase Mode, the human character is driven by an evasion algorithm based on Breadth-First Search (BFS). To address the core task of predator-prey pursuit, the project implements four distinct agent strategies: 

\begin{itemize}
    \item \textbf{A* Search Algorithm:} Delivers optimal pathfinding by calculating the shortest route to the human in single-monster mode. In two-monster mode, the system dynamically divides labor between a "direct chaser" and a "toroidal interceptor," with the latter predicting and cutting off the human's evasion route.
    
    \item \textbf{Greedy Algorithm:} Leverages a precomputed distance field centered on the human's current position. The monster blindly advances toward the neighboring node with the lowest distance value, achieving high-efficiency pursuit at a minimal computational cost.
    
    \item \textbf{Minimax Decision Tree:} Formulates the pursuit as a two-player adversarial game. By integrating $\alpha-\beta$ pruning with a transposition table, this module conducts multi-round forward look-ahead searches to minimize the human's safety score while simulating the human's attempts to maximize it, thereby achieving intelligent, coordinated multi-monster encirclement.
    
    \item \textbf{Simulated Annealing Algorithm:} Generates an initial baseline path and subsequently applies random perturbations over successive iterations. Guided by a cooling schedule, the algorithm accepts suboptimal movement decisions with a certain probability, thereby generating more stochastic and unpredictable monster behaviors to enhance gameplay engagement.
\end{itemize}
The central environment of our project is a $30 \times 30$ toroidal grid map, where the maze layouts are generated by a highly customized Genetic Algorithm (GA). This algorithm maintains a population of grid candidates and drives their evolution over 20 generations. A multi-objective fitness function carefully balances four core metrics to ensure the generation of high-quality mazes，including high floor-cell connectivity, a target wall density of approximately 28\%, a high proportion of three-way and four-way junction nodes, and an average BFS path length approaching 20 steps between randomly sampled cell pairs. Driven by tournament selection, row/column crossover operators, and bit-flip mutation, the algorithm converges efficiently to consistently output high-quality mazes characterized by high accessibility, spatial diversity, and moderate difficulty.

This report will elaborately present the design and concrete implementation of each component, quantitatively analyze the algorithmic properties of each AI strategy along with their impact on gameplay, and reflect on the respective strengths and limitations observed during development.

\section{Methodology}

\subsection{Map Representation}

The game map is represented as a 30 by 30 two-dimensional grid. In the map, 0 represents a walkable floor cell, while 1 represents a wall. The human and monsters can only move on walkable cells.

The map also uses a wrap-around boundary system. For example, if a character moves left from column 0, it will appear at column 29. This is implemented by using the modulo operation.

\subsection{Genetic Algorithm for Map Generation}

A Genetic Algorithm is used to generate playable maps. Each map is treated as one chromosome. A chromosome is represented as a 30 by 30 binary grid, where 0 means floor and 1 means wall.

The initial population contains several randomly generated maps. Each map is evaluated using a fitness function. The fitness function considers reachability, wall ratio, junction complexity, and path distance. Reachability checks whether most walkable cells are connected. Wall ratio prevents the map from being too empty or too blocked. Junction complexity encourages more branching paths. Path distance helps keep the chase distance reasonable.

After fitness evaluation, tournament selection is used to choose better maps as parents. Crossover combines two parent maps by cutting and joining their rows or columns. Mutation randomly changes a small number of cells from floor to wall or from wall to floor. After several generations, the best map is saved as a text file using 0 and 1 values separated by spaces.

\subsection{Human Movement and BFS}

The human moves one step per turn. BFS can be used to search reachable cells and support escape path planning. Since the map is an unweighted grid, BFS is suitable for finding shortest paths.

\subsection{Monster AI Algorithms}
\subsubsection{A* Algorithm}
A* is an informed graph search algorithm that finds the shortest path between two nodes by combining the cost of the path taken so far with a heuristic estimate of the remaining cost to the goal. Formally, each node is evaluated by the function below, where $g(n)$ is the cost from the start to node $n$, and $h(n)$ is the heuristic estimate from $n$ to the goal.

\[
f(n) = g(n) + h(n)
\]

In this project, the A* algorithm is implemented in astar.py to drive the monster toward the player each turn. The grid is treated as an unweighted graph in which each walkable grid cell has up to four neighbors, reached by movement in the cardinal directions (up, down, left, and right). The heuristic function used is the Manhattan distance between the current cell and the player's position. The toroidal topology is handled by applying a modulo operation to row and column indices at every neighbor expansion, thereby naturally accommodating movement that wraps across the grid boundaries.The core function astar(grid, start, goal) returns the full path from the monster's current position to the player. The game logic then takes the second element of this path as the monster's next step, executing only one grid cell of movement per turn. If no path exists (for example, if the player is completely enclosed by walls), the monster stays in place.
\subsubsection{Double Monsters}
In two-monster A* mode, the algorithm assigns distinct roles to each monster to increase coordination and reduce the likelihood that both monsters converge on the same corridor. The monster currently closer to the player (measured by Manhattan distance) is designated as the chaser and pursues the player directly using A*. The farther monster is designated as the interceptor and moves toward an intercept point rather than the player's current position.
The intercept point is computed by projecting the player's position forward in the direction from the chaser to the player by a fixed offset of four cells, adjusted for the toroidal topology. If the projected cell is a wall, the intercept target falls back to the player's current position, causing the interceptor to also pursue directly. This simple role assignment means the two monsters approach from different angles, narrowing the player's escape options more effectively than two independent chasers would.

The key functions for this mode are get\_next\_step\_single, which returns the next cell for a single-monster chase, and get\_next\_steps\_two, which coordinates both monsters by determining which is the chaser and which is the interceptor before computing each monster's next step independently via A*.
\subsection{Item System}

Items are generated after a fixed number of human steps. For example, new items may appear every 10 human steps. Unused items remain on the map. Items can affect the movement and strategy of the game, such as giving the human a speed boost or temporarily stopping the monster.

\section{Implementation}

The project is divided into several modules. The AI folder stores the algorithm implementations, including Genetic Algorithm, BFS, Greedy Search, A*, Minimax, and Simulated Annealing. The game folder stores the main game logic, including map handling, player movement, monster movement, item logic, collision checking, and scoring.

The Genetic Algorithm module outputs a text file containing the generated map. The game system can then load this map and use it during gameplay.

\section{Validation and Verification}

Several tests are used to validate the project.

\subsection{Map Generation Test}

The generated map should have 30 rows and 30 columns. Each cell should only contain 0 or 1. The wall ratio should be reasonable, and most walkable cells should be connected.

\subsection{Spawn Test}

The human and monster should spawn on walkable cells. They should not spawn on the same position. Their starting distance should also be within a reasonable range to avoid unfair gameplay.

\subsection{Boundary Test}

The wrap-around boundary is tested by moving characters across the edges of the map. Moving left from column 0 should move the character to column 29. Moving up from row 0 should move the character to row 29.

\subsection{Algorithm Comparison}

The monster AI algorithms can be compared using capture steps, runtime, and behavior. Greedy Search is simple and fast. A* is more stable for pathfinding. Minimax can predict future movement but requires more computation. Simulated Annealing provides more random behavior.

\section{Results and Discussion}

The project successfully generates playable maps and supports AI-based monster chasing behavior. The Genetic Algorithm improves map quality compared with purely random map generation. BFS can support human escape planning. A* provides more reliable chasing behavior than Greedy Search when obstacles exist.

The project also shows that different algorithms have different strengths and weaknesses. Simple algorithms are easier to run but may perform poorly in complex maps. More advanced algorithms can make better decisions but may require more computation.

\section{Conclusion}

In conclusion, this project demonstrates how artificial intelligence algorithms can be applied in an interactive survival chase game. The Genetic Algorithm is used for map generation, BFS is used for human path searching, and several monster AI algorithms are used for chasing behavior. Through this project, the team gained practical experience in AI search algorithms, optimization, game logic, and system integration.

Future improvements may include better multi-agent monster cooperation, more advanced item effects, improved user interface design, and reinforcement learning-based monster behavior.
