\section*{Abstract}

Our project is a survival game called \textbf{Maze Escape}. The goal of the project is to show the practical use of artificial intelligence search and optimization algorithms. The game simulates humans being chased by monsters on a 30 $\times$ 30 circular grid map (wrapping edges). The system uses and compares five AI search strategies: \textbf{A* pathfinding},\textbf{ Greedy Algorithm}, \textbf{breadth-first search (BFS)}, \textbf{Minimax Algorithm}, and \textbf{Simulated Annealing Algorithm}. The maze is generated by a \textbf{Genetic Algorithm (GA)}. The GA improves map connectivity, wall density, path diversity, and branch structure through multi-generation evolution.

This project shows how artificial intelligence behaves under limited, competitive, and real-time conditions. It also shows how different search strategies change the difficulty of the game and affect the player experience. The A* algorithm creates the most direct pursuit path. The greedy algorithm gives fast performance with low computing cost. The simulated annealing algorithm adds controlled randomness and makes monster movement less predictable. The genetic algorithm keeps generating connected and navigable maps to maintain gameplay quality.

In the SOF Principle of Artificial Intelligence course, we explored a variety of search, planning, and optimization algorithms. Built upon these foundational course concepts, our Maze Escape project elevates static, single-agent pathfinding into a dynamic, multi-dimensional adversarial scenario by introducing procedurally evolving mazes, expanding variable dimensions, and incorporating bidirectional pursuit dynamics.

To comprehensively evaluate algorithmic behaviors, we designed two complementary gameplay modes: Escape Mode and Chase Mode. In Chase Mode, the human character is driven by an evasion algorithm based on Breadth-First Search (BFS). To address the core task of predator-prey pursuit, the project implements four distinct agent strategies: 

\begin{itemize}
    \item \textbf{A* Search Algorithm:} In single-monster mode, the algorithm finds the shortest path to the human player. In dual-monster mode, the system divides the monsters into a “direct pursuer” and an “interceptor.” The interceptor predicts future movement and blocks escape routes.
    
    \item \textbf{Greedy Algorithm:} The algorithm calculates the distance field by using the player’s current position as the center. The monster always moves to the neighboring node with the smallest distance value. This gives efficient pursuit with very low computational cost.
    
    \item \textbf{Minimax Decision Tree:} The chase process is treated as a two-player game. The system combines $\alpha-\beta$ pruning and a transposition table to search several future turns. The monster tries to reduce the human safety score, while the simulated human tries to increase it. This creates coordinated multi-monster pursuit behavior.
    
    \item \textbf{Simulated Annealing Algorithm:} The algorithm first generates an initial path and then randomly changes it during repeated iterations. Based on a cooling schedule, the system sometimes accepts worse movement decisions with a certain probability. This creates more random and less predictable monster behavior.
\end{itemize}
The central environment of our project is a $30 \times 30$ toroidal grid map, where the maze layouts are generated by a highly customized Genetic Algorithm (GA). The algorithm keeps a population of candidate maps and evolves them for more than 20 generations. The fitness function balances several important factors, including strong floor connectivity, about 28\% wall density, a high number of three-way and four-way junctions, and an average BFS path length close to 20 steps between randomly selected positions. The system also uses tournament selection, row and column crossover, and bit-flip mutation. These methods help the algorithm generate maps with good accessibility, spatial diversity, and suitable difficulty.

The following report will introduce the design and specific implementation methods of each component in detail, quantitatively analyze the algorithm characteristics of each artificial intelligence strategy and its impact on the gameplay, and reflect on the respective advantages and limitations observed in the development process.

\section{Methodology}

\subsection{Map Representation}

The game map is represented as a 30 by 30 two-dimensional grid. In the map, 0 represents a walkable floor cell, while 1 represents a wall. The human and monsters can only move on walkable cells.

The map also uses a wrap-around boundary system. For example, if a character moves left from column 0, it will appear at column 29. This is implemented by using the modulo operation.

\subsection{Genetic Algorithm for Map Generation}

A Genetic Algorithm is used to generate playable maps. Each map is treated as one chromosome. A chromosome is represented as a 30 by 30 binary grid, where 0 means floor and 1 means wall.

The initial population contains several randomly generated maps. Each map is evaluated using a fitness function. The fitness function considers reachability, wall ratio, junction complexity, and path distance. Reachability checks whether most walkable cells are connected. Wall ratio prevents the map from being too empty or too blocked. Junction complexity encourages more branching paths. Path distance helps keep the chase distance reasonable.

After fitness evaluation, tournament selection is used to choose better maps as parents. Crossover combines two parent maps by cutting and joining their rows or columns. Mutation randomly changes a small number of cells from floor to wall or from wall to floor. After several generations, the best map is saved as a text file using 0 and 1 values separated by spaces.

\subsection{Human Movement and BFS}

The human moves one step per turn. BFS can be used to search reachable cells and support escape path planning. Since the map is an unweighted grid, BFS is suitable for finding shortest paths.

\subsection{Monster AI Algorithms}
\subsubsection{A* Algorithm}
A\* is an informed graph search algorithm. It finds the shortest path between two nodes. It does this by adding the cost of the path taken so far to an estimate of the remaining cost to the goal. Each node is scored by the function below，where $g(n)$ is the cost from the start to node $n$, and $h(n)$ is the heuristic estimate from $n$ to the goal.

\[
f(n) = g(n) + h(n)
\]

In this project, the A* algorithm is used in astar.py. It moves the monster toward the player each turn. The grid is treated as an unweighted graph. Each walkable grid cell has up to four neighbors. These neighbors are reached by moving up, down, left, or right. The heuristic function used is the Manhattan distance between the current cell and the player's position. The toroidal topology is handled by applying a modulo operation to row and column indices at every neighbor expansion. This lets movement wrap across the grid boundaries.
The core function astar(grid, start, goal) returns the full path from the monster's current position to the player. The game logic then takes the second element of this path as the monster's next step. Only one grid cell of movement happens per turn. If no path exists, for example if the player is fully enclosed by walls, the monster stays in place.

\subsubsection{Double Monsters}
In two-monster A* mode, the algorithm gives each monster a different role. This reduces the chance that both monsters go to the same corridor. The monster closer to the player, measured by Manhattan distance, is the chaser. It goes after the player directly using A*. The farther monster is the interceptor. It moves toward an intercept point instead of the player's current position.

The intercept point is found by projecting the player's position forward in the direction from the chaser to the player by a fixed offset of four cells. This is adjusted for the toroidal topology. If the projected cell is a wall, the intercept target falls back to the player's current position. In that case, the interceptor also goes after the player directly. This role assignment means the two monsters come from different angles. This makes it harder for the player to escape. The key functions for this mode are get\_next\_step\_single, which returns the next cell for a single-monster chase, and get\_next\_steps\_two, which sets the role of each monster before computing each monster's next step using A*.
\subsection{Item System}

Items are generated after a fixed number of human steps. For example, new items may appear every 10 human steps. Unused items remain on the map. Items can affect the movement and strategy of the game, such as giving the human a speed boost or temporarily stopping the monster.

\section{Implementation}

The project is divided into several modules. The AI folder stores the algorithm implementations, including Genetic Algorithm, BFS, Greedy Search, A*, Minimax, and Simulated Annealing. The game folder stores the main game logic, including map handling, player movement, monster movement, item logic, collision checking, and scoring.

The Genetic Algorithm module outputs a text file containing the generated map. The game system can then load this map and use it during gameplay.

\section{Validation and Verification}

Several tests are used to validate the project.

\subsection{Map Generation Test}

The generated map should have 30 rows and 30 columns. Each cell should only contain 0 or 1. The wall ratio should be reasonable, and most walkable cells should be connected.

\subsection{Spawn Test}

The human and monster should spawn on walkable cells. They should not spawn on the same position. Their starting distance should also be within a reasonable range to avoid unfair gameplay.

\subsection{Boundary Test}

The wrap-around boundary is tested by moving characters across the edges of the map. Moving left from column 0 should move the character to column 29. Moving up from row 0 should move the character to row 29.

\subsection{Algorithm Comparison}

The monster AI algorithms can be compared using capture steps, runtime, and behavior. Greedy Search is simple and fast. A* is more stable for pathfinding. Minimax can predict future movement but requires more computation. Simulated Annealing provides more random behavior.

\section{Results and Discussion}

The project successfully generates playable maps and supports AI-based monster chasing behavior. The Genetic Algorithm improves map quality compared with purely random map generation. BFS can support human escape planning. A* provides more reliable chasing behavior than Greedy Search when obstacles exist.

The project also shows that different algorithms have different strengths and weaknesses. Simple algorithms are easier to run but may perform poorly in complex maps. More advanced algorithms can make better decisions but may require more computation.

\section{Conclusion}

In conclusion, this project demonstrates how artificial intelligence algorithms can be applied in an interactive survival chase game. The Genetic Algorithm is used for map generation, BFS is used for human path searching, and several monster AI algorithms are used for chasing behavior. Through this project, the team gained practical experience in AI search algorithms, optimization, game logic, and system integration.

Future improvements may include better multi-agent monster cooperation, more advanced item effects, improved user interface design, and reinforcement learning-based monster behavior.
